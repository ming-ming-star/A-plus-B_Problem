# IdealVM

一个轻量级的自定义指令集虚拟机，使用 C++ 实现。支持基本算术运算、位运算、条件跳转与 I/O 操作，能在一个接近裸机的抽象层上运行程序。


## 快速开始

将代码编译后直接运行即可。虚拟机启动时会从 code 数组中逐条读取并执行指令，无需额外输入程序文件。

```bash
g++ -std=c++14 -O2 main.cpp -o idealvm
./idealvm
```

程序执行后将按照 code 数组中的指令自动完成输入和输出。


## 指令集

虚拟机配备 1,048,576 个内存单元，每个单元存储一个有符号 64 位整数。指令均以字符串形式定义，操作数之间以空格分隔。

数据读写

|指令 |	语法 | 说明|
|:-:|:--------:|:--:|
|`input` | `input {x}` | 从标准输入读入一个整数，存入内存 `x` 号位置
|`output`	| `output {mode} {x}`	| 以指定模式输出内存 `x` 号位置的值。`mode` 可选 `int`（十进制整数）或 `char`（按 ASCII 码输出字符）
|`write`	| `write {x} {y}`	| 将内存 `x` 号位置设为常量 `y`
|`copy`	| `copy {x} {y}` | 将内存 `x` 号位置的值拷贝到内存 `y` 号位置

算术与位运算

|指令	| 语法 | 说明
|:-:|:--------:|:--:|
|`calc`	| `calc {mode} {x} {y} {z}`	| 将内存 `x` 与 `y` 的值按 `mode` 运算，结果存入内存 `z`

`mode` 可选值：

|模式	| 运算类型	| 含义
|:-:|:--------:|:--:|
|`add` | 算术 | `x + y`
|`sub` | 算术 | `x - y`
|`mul` | 算术 | `x * y`
|`div` | 算术 | `x / y`（除数为零时报错退出）
|`mod` | 算术 | `x % y`
|`and` | 位运算 | 	`x & y`
|`or` | 位运算 | `x \| y`
|`xor` | 位运算 | `x ^ y`
|`not` | 位运算 | `~x`，结果存入 `y`（仅需两个操作数）

控制流

指令 | 语法 | 说明
|:-:|:--------:|:--:|
|`goto` | `goto {x}` | 无条件跳转至第 `x` 行（行号从 0 开始）
|`if` | `if {mode} {x} {y} {z}` | 比较内存 `x` 与 `y` 的值，若关系 `mode` 成立则跳转至第 `z` 行
|`++` | `++ {x}` | 将内存 `x` 号位置的值加 1
|`--` | `-- {x}` | 将内存 `x` 号位置的值减 1
|`exit` | `exit` | 终止程序

`if` 的 `mode` 可选：`<`、`>`、`==`、`<=`、`>=`、`!=`。

## 示例：A+B Problem
``` cpp
input 0            # 读入第一个整数，存入内存 0
input 1            # 读入第二个整数，存入内存 1
calc add 0 1 2     # 计算 a + b，结果存入内存 2
output int 2       # 以整数形式输出结果
exit               # 退出程序
```

对应的 `code` 数组配置：

```cpp
string code[] = {
    "input 0",
    "input 1",
    "calc add 0 1 2",
    "output int 2",
    "exit"
};
```

运行后输入 `20 30`，程序输出 `50`。


## 数据范围

* 整数范围：[-2⁶³, 2⁶³]
* 内存大小：1,048,576 个单元
* 输入输出默认以空格或换行分隔

错误处理

* 除数为零时输出 `Error: Division by Zero` 并退出
* 遇到无效指令时输出 `Error: Invalid Code` 并退出


## 设计理念

这个虚拟机最初是为了完成一道 A+B Problem 而诞生的——用“先造一台计算机，再跑加法”的方式诠释了“任何一个伟大的思想，都有一个微不足道的开始”。它刻意保持了最小化的指令集，在约束中探索图灵完备的计算边界，适合用于理解计算机体系结构的基本原理。
